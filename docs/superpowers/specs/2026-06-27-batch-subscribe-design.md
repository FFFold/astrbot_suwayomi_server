# 批量订阅功能设计

**日期**: 2026-06-27
**状态**: 已批准

## 背景

当前订阅漫画需要逐个搜索再订阅，效率低。用户希望一次性提供多个漫画名称，插件自动搜索并批量订阅。

## 命令语法

```
漫画 批量订阅 <名称1>, <名称2>, <名称3>, ... [源名]
```

源名可选，放在最后，与现有 `「漫画 搜索 <关键词> [源名]」` 模式一致。

分隔符支持：`,`, `，`, `;`, `；`

示例：
```
漫画 批量订阅 咒术回战, 鬼灭之刃, 电锯人
漫画 批量订阅 咒术回战, 鬼灭之刃, 电锯人 jm
```

## 执行流程

```
输入: "咒术回战, 鬼灭之刃, 电锯人"

1. 解析出名称列表: ["咒术回战", "鬼灭之刃", "电锯人"]
2. 发送进度提示: "📚 开始批量订阅 3 部漫画..."
3. 逐个处理:
   [1/3] 搜索「咒术回战」→ 找到「咒術廻戦」→ 订阅成功
   [2/3] 搜索「鬼灭之刃」→ 找到「鬼滅の刃」→ 订阅成功
   [3/3] 搜索「电锯人」→ 未找到 → 跳过
4. 最终汇总:
   "✅ 批量订阅完成 (2/3 成功):
    ✅ 咒術廻戦 - 连载中 - 拷贝漫画 (ZH)
    ✅ 鬼滅の刃 - 已完结 - 禁漫天堂 (JM)
    ❌ 电锯人 - 未找到匹配结果"
```

## 汇总消息格式

每条结果包含：
- 状态图标（✅ 已订阅 / ⏭ 已存在 / ❌ 失败）
- 漫画标题（来自搜索结果）
- 连载状态（连载中/已完结等）
- 源名称

## 错误处理

- 搜索无结果 → 记录失败，继续处理下一个
- 已订阅 → 记录为"已存在"，不重复订阅
- 网络/API 错误 → 记录失败，继续
- 源不存在 → 整体报错

## 数量限制

最多 20 个漫画名称，防止滥用。

## 实现方案

**改动范围**：仅修改 `main.py`，新增约 60-80 行代码。

**新增命令方法**：`batch_subscribe(self, event)`

**新增辅助方法**：`_search_best_match(self, name, source_filter)` — 复用现有搜索逻辑，返回第一个匹配结果。

**复用现有组件**：
- `self.client.search_manga()` — 搜索 API
- `self.sub_mgr.subscribe()` — 订阅存储
- `self._get_or_fetch_chapters()` — 获取章节
- `self.sub_mgr.update_latest_chapter()` — 更新最新章节 ID

**解析逻辑**：
1. 从 `event.message_str` 获取原始输入
2. 检查末尾是否匹配源名/语言/ID（复用搜索命令的源匹配逻辑）
3. 剩余部分按逗号/分号分割
4. 去除空白，过滤空串
5. 校验数量 ≤ 20

**关键参考**：
- AGENTS.md 第 12 条：AstrBot 按空格分割参数，必须从 `event.message_str` 解析
- 现有 `search_manga` 命令（main.py:270）的源匹配逻辑
- 现有 `subscribe_manga` 命令（main.py:345）的订阅逻辑

## 不做的事情

- 不支持从文件导入列表
- 不支持并行搜索（保持简单可靠）
- 不支持两阶段确认（自动匹配最佳结果）
- 不修改现有搜索/订阅命令
